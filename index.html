Tech Store 3D Website

I built this 3D web application as my assignment for the Web3D Applications module at the University of Sussex. The project began in January when I set up my development environment in Lab 1 using Visual Studio Code and the Live Server extension, and learned the basics of structuring a web page with HTML, CSS, and Bootstrap. From there I progressively developed the project across eleven labs - moving through 3D modelling in Blender, integrating models into a Three.js web page, adding interactivity with JavaScript, and finally refining and testing the finished application in April.

Rather than following the Coca-Cola brand from the labs, I chose to design an original tech store concept with four products I modelled entirely myself in Blender: a laptop, a vertical-fold phone, an earbuds case, and a horizontal-fold phone.

Development Journey Through the Labs

  • January - Lab 1: I set up VS Code with the Live Server extension and built my first HTML/CSS web page using Bootstrap. I familiarised myself with the module brief and marking rubric, and looked at examples on the Three.js website to understand what the final project could look like.
  • February - Labs 2-3: In Lab 2 I continued working with Bootstrap's grid and layout system to plan my page structure. In Lab 3 I started learning Blender, where the lab covered how to model a Coca-Cola can, apply materials, and create an animation. I used these skills as a foundation but chose to model my own tech products instead.
  • February - Labs 4-5: Lab 4 taught me how to use Three.js to add a 3D model to a web page - setting up a Scene, Camera, and Renderer, and loading a GLB file with GLTFLoader. Lab 5 introduced wireframe mode, model switching, adding audio, and a lighting GUI. I applied all of these concepts directly to my own project.
  • March - Labs 6 to 8: Labs 6 to 8 were focused on building and refining my web pages and models. I used this time to build all four of my Blender models, add them to my web page, and implement the JavaScript interactivity. I also applied the MVC framework pattern from the lab resources to separate my code into distinct modules.
  • April - Labs 9 to 11: In Lab 9 I focused on the visual design and CSS polish. Lab 10 was dedicated to building the About page and adding evidence. In Lab 11 I carried out structured testing in both Chrome and Safari using the debugging tools covered in the lab, and recorded the results in the testing table below.

The Four 3D Models

Each model has three colour finishes you can explore. Select a model from the sidebar, then use the colour swatches in the panel on the right side of the 3D viewport to switch finishes in real time.

  • Laptop Pro - I designed this clamshell laptop in Blender with an aluminium body, keyboard, trackpad, hinge cylinders, and a lit display. The screen lid opens from closed to 110° via the Animate button. Available finishes: Silver, Space Grey, Rose Gold.
  • Flip Phone Pro - I modelled this vertical-fold phone in Blender with a hinge band, triple rear camera array on the back of the top half, and all physical buttons on the right side of the bottom half. The top half folds to 175°. Available finishes: Midnight, Pearl, Lavender.
  • EarPods Pro - I built this rounded earbuds charging case in Blender, complete with a hinged lid, LED status indicator, USB-C port, and two earbuds with silicone tips visible inside when open. Lid opens to 112°. Available finishes: White, Midnight, Yellow.
  • Fold Phone Pro - I designed this book-fold phone in Blender with two screen panels joined by a spine strip, hinge detail lines, and a triple rear camera array. The left panel unfolds to 175°. Available finishes: Phantom, Icy Blue, Burgundy.

Interactivity & JavaScript

  • Animate - animProg (0-1) tweened per frame, lerped between open/closed Euler rotations on the named pivot group; sound plays simultaneously
  • Wireframe - Traverses all GLTF meshes and toggles wireframe on every material from Lab 5
  • Lighting presets - Cycles Studio, Neon, Daylight; zeroes all lights before applying preset which came from Lab 5
  • Spotlight - SpotLight independently toggleable; gold button when active
  • Camera presets - Cubic ease-out camera lerp to four named positions each frame
  • Colour variants - Reads hex from products.json; lerps material colour at 0.48 blend; emissive surfaces skipped
  • GLSL rim shader - Toggle on/off; RIM COLOUR cycles uRimColor uniform live on GPU
  • Video panel - HTML5 video source set from JSON data; pauses on close
  • Content swapping - Sidebar click swaps GLB, specs, thumbnail, description, and swatches simultaneously
  • JSON/AJAX boot - fetch('data/products.json') resolves before app initialises

Lighting & Camera

  • Studio - AmbientLight + DirectionalLight key (shadows) + blue fill. Professional product photography balance.
  • Neon - Cyan, magenta, green PointLights. Dramatic coloured rim lighting.
  • Daylight - Warm DirectionalLight + HemisphereLight. Outdoor feel.
  • Spotlight - Focused warm SpotLight from above, independently toggleable.
  • ACES Filmic tone mapping (1.55) + sRGB encoding. PCFSoft shadow maps.
  • 42° FOV camera; damped OrbitControls; exponential fog.

Media Integration

  • 3D Models (GLB) - Four models created in Blender, loaded asynchronously via GLTFLoader; loading spinner shown during fetch
  • Video (HTML5) - VIDEO button opens an overlay with a native <video controls> element loaded from assets/video/[id].mp4; graceful fallback if file absent
  • Images (Canvas API) - I used the HTML5 Canvas 2D API to draw a product silhouette and convert it to a data URL, which is set as the <img> src in the specs panel - no image files needed
  • Audio (Web Audio API) - Ambient two-oscillator sine drone, sawtooth whoosh on Animate, sine click on product switch - all procedurally synthesised without audio files. This builds on the audio covered in Lab 5.
  • Evidence images - Screenshots of Blender model building and browser testing displayed in this About page as visual evidence of the development process
  • GLSL Shader - Custom ShaderMaterial with vertex and fragment shaders renders a rim/glow halo; rim colour uniform updated live on the GPU
  • JSON Data - Product descriptions, specs, colours, and video paths loaded from products.json via fetch() and displayed dynamically

Accessibility

  • Keyboard nav - All buttons are native HTML elements; Tab-focusable and Enter/Space operable

  • Motion opt-in - Floating idle and auto-spin are off by default

  • Video controls - Native HTML5 controls; keyboard accessible
